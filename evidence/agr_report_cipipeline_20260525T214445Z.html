Zero secret-scanning steps exist in this pipeline across all executions. The policies_harness branch commit e1b2a38 (message: tflint from lsdeqaint harbor) was not scanned for secrets before build. Any hardcoded credentials, API keys, or tokens in the repository would be undetected by this pipeline and could be baked into published artifacts or infrastructure configurations. This is the highest residual risk item. Framework: SOC2 CC6.1/CC6.7 | NIST IA-5/SC-28 | cATO 1.0 CONMON | PCI 8.2/8.6
No SBOM — Zero Dependency Visibility for Shipped Artifacts
No SBOM was generated for any artifact produced by this pipeline. For container-producing executions, images are pushed to Harbor without a software bill of materials. This means: (1) No component inventory for vulnerability correlation; (2) No prohibited-package enforcement; (3) No SLSA provenance attestation. The ci-static-analysis script generates a local sbom.json report but this is not a formal SBOM artifact and is not attached to the image digest. Framework: SOC2 CC7.1/CC7.2 | NIST SR-4/SR-3 | SLSA L1 | cATO 3.1 | PCI 6.3.2
OPA On-Save Gates Disabled — Structural Violations Not Caught at Author Time
Three On-Save policy sets are present in the project but all are disabled: gate_pipeline_structure, gate_approved_environments, gate_integration. A pipeline author could remove required controls (secret scan, approval gate, SBOM step) and the change would not be blocked until the next execution — which may succeed if the removed control was the only blocking gate. Framework: SOC2 CC8.1 | NIST CM-3/CM-4 | cATO 3.2 | PCI 6.4
Custom Script Scanning — No STO Dashboard Visibility or Threshold Enforcement
All security scanning (SonarQube SAST, OSV SCA, Checkov IaC, Trivy container) runs via custom shell scripts in Run steps. Findings are written to /harness/ci-reports/ and summarized into OPA policy payloads. This architecture means: (1) No findings appear in the Harness STO Security Tests dashboard; (2) fail_on_severity cannot be configured on individual scanners; (3) Scan results depend on script correctness and field naming; (4) No centralized exemption/waiver management. Framework: SOC2 CC7.2/CC7.3 | NIST RA-5/SI-2 | cATO 3.2
