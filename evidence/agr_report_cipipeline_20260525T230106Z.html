EXECUTION ANALYSIS
Secret Detection Blind Spot — Every Execution
With no Gitleaks or secrets scanner in the pipeline, any API key, private key, service account credential, or database password committed to the policy-as-code repo ships through every execution undetected. Run #21 processed commit e1b2a38 from branch policies_harness — any secrets in that commit are now in the built artifact. Framework mapping: SOC2 CC6.1/CC6.7 | NIST IA-5/SC-28 | cATO 1.0 | PCI 8.2/8.6
SBOM Chain-of-Custody Gap — Provenance Unverifiable
The pipeline generates SBOM data via a shell script inside a Run step. Script output can be tampered between generation and consumption, has no cryptographic binding to the built artifact, and is not registered in the Harness SCS module. The OPA gate evaluates artifact.sbom_present from this unverified source. In Run #21 (terraform), sbom_present: false because the package stage was skipped. Framework mapping: SOC2 CC7.1 | NIST SR-4/SA-11 | SLSA L1 | cATO 3.1 | PCI 6.3.2
Signing Stage Skipped — Run #21 Terraform Artifact Unsigned
The CI_Sign stage was SKIPPED in Run #21 because language=terraform triggers the skip condition. Terraform modules are distributable artifacts and require the same provenance chain as container images, particularly for IaC applied to production infrastructure. The signing implementation also relies on fetching ci/sign/ci-sign from GitLab at runtime — if the GitLab host is unreachable or the repo is compromised, signing silently fails. Framework mapping: SOC2 CC6.1 | NIST SI-7/SA-12 | SLSA L2-L3 | cATO 3.1 | PCI 6.2
Build Tool Floating Tags — Trivy and ZAP Images Unpinned
Both aquasec/trivy:latest and zaproxy/zap-stable:latest are security-critical scanning tools used with floating latest tags. If these images are compromised in the Harbor registry mirror (which proxies external images), the scanners themselves become attack vectors. A compromised Trivy could suppress critical CVE findings without any pipeline alarm. Framework mapping: SOC2 CC7.1/CC8.1 | NIST SA-12/SR-3/SR-4 | SLSA L3 | PCI 6.2
Publish-Reports Script Not Found — Evidence Export Silently Skipped
In Run #21, the publish_reports_static step debug logs confirm: 'ERROR: publish-reports script not found anywhere'. The failureStrategies: Ignore on all publish steps means governance report publication failures are invisible in pipeline status. No error, no alert. The governance GitLab repository may be missing critical evidence for this execution. Framework mapping: SOC2 CC4.1/CC7.4 | NIST CA-2/CA-7(1) | SLSA L2 | cATO 1.0 | PCI 12.4
OPA Gate Payload Coverage Incomplete — Script Crash Bypasses Gate
STO threshold enforcement relies on OPA gates consuming JSON reports generated by shell scripts. If a script crashes or produces malformed output, the step uses default empty values (critical_cves: 0, scan_completed: false) which pass the policy gate. There is no native STO fail_on_severity safety net. This is a silent false-negative vector: scan failures look like clean scans. Framework mapping: SOC2 CC7.2/CC7.3 | NIST RA-5/SI-2 | cATO 3.2 | PCI 6.3.1/11.3
